Collections Java API 2024

Repositório de estudos do Bootcamp Java Backend Santander — DIO. Exercícios práticos sobre a API de Collections do Java.

Sobre o projeto

Este repositório reúne 24 classes Java organizadas para aprender o uso de List, Set e Map — os três pilares da Java Collections Framework.

Cada exercício é uma classe independente com método main, permitindo executar e testar os conceitos diretamente no IDE.

Java 17 IntelliJ IDEA Sem Maven/Gradle GPL-3.0

Estrutura do repositório

src/main/java/
├── list/          → ArrayList
│   ├── OperacoesBasicas/
│   ├── Pesquisa/
│   └── Ordenacao/
├── set/           → HashSet, TreeSet
│   ├── OperacoesBasicas/
│   ├── Pesquisa/
│   └── Ordenacao/
└── map/           → HashMap
    ├── OperacoesBasicas/
    └── Pesquisa/

A organização segue o padrão: tipo de coleção × tipo de operação.

Módulos e exemplos

List — ArrayList

  • Operações básicas: CarrinhoDeCompras, ListaTarefas, Tarefas
  • Pesquisa: CatalogoLivros, SomaNumeros
  • Ordenação: OrdenacaoPessoa, OrdenacaoNumero

Set — HashSet / TreeSet

  • Operações básicas: ConjuntoConvidados, ConjuntoPalavrasUnicas
  • Pesquisa: AgendaContatos, ListaTarefas
  • Ordenação: CadastroProdutos, GerenciadorAlunos

Map — HashMap

  • Operações básicas: AgendaContatos (nome → telefone)
  • Pesquisa: EstoqueProdutos (valor total, produto mais caro)

Conceitos abordados

Conceito Onde aparece
ArrayList Listas ordenadas, permite duplicatas
HashSet Conjuntos sem duplicatas (equals / hashCode)
TreeSet Conjuntos ordenados (Comparable / Comparator)
HashMap Pares chave-valor
Comparable Ordenação natural (Pessoa, Produto)
Comparator Ordenação customizada (altura, preço, nota)

Como executar

  1. Clone o repositório e abra no IntelliJ IDEA (Java 17).
  2. Navegue até qualquer classe com public static void main.
  3. Execute com o botão Run ou Shift + F10.

Exemplo: src/main/java/list/OperacoesBasicas/CarrinhoDeCompras.java

Observações técnicas

  • Pacotes no formato main.java.list... (estrutura do bootcamp, não padrão Maven).
  • Sem testes automatizados — validação feita pelos métodos main.
  • Algumas classes compartilham nomes em pacotes diferentes (ex.: AgendaContatos em map e set).
